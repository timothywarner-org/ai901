================================================================================
 STEP 2 -- Structured Outputs with JSON Schema
   Same triage scenario -- now the format is GUARANTEED, not just demonstrated.
================================================================================

--------------------------------------------------------------------------------
 [2a] Triage system message (paste, Apply changes)
      Same persona as Step 1, restated for a clean start.
--------------------------------------------------------------------------------
You are a support-ticket triage assistant for Contoso Outdoor. Return only JSON conforming to the schema. Safety issues and anything mentioning injury are always high priority and always need a human.


--------------------------------------------------------------------------------
 [2b] triage_result JSON schema
      (Response format dropdown -> JSON schema -> name it triage_result,
       paste this, toggle Strict ON, Save)
--------------------------------------------------------------------------------
{
  "type": "object",
  "properties": {
    "category": { "type": "string", "enum": ["product_defect", "shipping_question", "billing", "returns", "general"] },
    "priority": { "type": "string", "enum": ["low", "medium", "high"] },
    "needs_human": { "type": "boolean" }
  },
  "required": ["category", "priority", "needs_human"],
  "additionalProperties": false
}


--------------------------------------------------------------------------------
 [2c] User ticket to fill the schema
      A safety ticket -- shows the system rule + schema combine to escalate.
--------------------------------------------------------------------------------
I followed the setup video but the stove regulator hisses and I can smell gas. Is this safe to use on my trip this weekend?

================================================================================
 POCKET CARDS -- reference while running the demo (do not paste into the portal)
================================================================================

--------------------------------------------------------------------------------
 [PC-1] Structured Outputs three-rule pocket card (Step 2)
--------------------------------------------------------------------------------
Structured Outputs requires THREE non-negotiable schema rules:

  1. Every field you want in the output MUST be listed in required[]
  2. additionalProperties MUST be set to false on every object
  3. Schema can have at most 5 levels of nesting and 100 properties total

Plus: strict: true is NOT optional. Without it, you fall back to JSON mode
with no schema guarantees.

Minimum api_version: 2024-08-01-preview. On a GA stack, 2024-10-21 (what the
SDK bookend and the rest of this course use) carries the same guarantee.


--------------------------------------------------------------------------------
 [PC-2] Prompt Shields attack-class pocket card (Step 3)
--------------------------------------------------------------------------------
Four Prompt Shields attack classes the exam expects you to name:

  1. Change system rules     -- "ignore all previous instructions..."
  2. Conversation mockup     -- attacker fakes an assistant turn
  3. Role-play persona swap  -- "you are now FreeBot / DAN / GodMode"
  4. Encoding attacks        -- instructions hidden in base64 / unicode

Prompt Shields runs deterministic classifiers BEFORE the model sees the
prompt. System message is guidance; Prompt Shields is the control.

GA since August 2024.
